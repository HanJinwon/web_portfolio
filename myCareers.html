웹 프론트엔드 개발

AnyLink의 Web Edition 중 프론트엔드 단을 개발하였습니다 (2019.03 ~ 2019.06)

TmaxSoft에서의 첫 프로젝트

  • TypeScript 언어와 React 라이브러리, 그리고 Theia 프레임워크를 사용하여 개발하였습니다.
  • 기존 AnyLink 7 Studio 제품의 UI/UX를 분석하여 웹 화면을 설계하였습니다.
  • 학부 시절 웹 개발 경력은 많지 않았지만 (거의 전무했지만) 새 언어를 습득하고 당해 2분기까지 맡은 부분에 대한 개발을 완료하였습니다.

웹 백엔드 개발

AnyLink의 Web Edition 중 백엔드 단을 개발하였습니다 (2019.07 ~ 2019.12)

프론트엔드에서 백엔드로

  • 제가 개발했던 AnyLink Web Edition의 Front End (Client)로부터 호출되는 API 서비스를 설계 및 개발하였습니다.
  • 설계한 API로는 프로젝트 형상관리 시스템 (SCM), 리소스 형상관리 시스템, 빌드 및 배포 요청에 대한 서비스입니다.
  • Spring Framework와 유사한 아키텍처인 ProObject와 Java언어, DB는 Tibero 6를 사용하여 개발하였습니다.
  • 클라이언트와 REST API를 주고 받으며, 프로젝트 형상 정보와 리소스 정보를 DB에 관리하고, Jenkins를 통한 빌드 및 엔진 배포 서비스를 개발하였습니다.
  • 당해 4분기까지 1차 릴리즈를 완료하였습니다.

Kubernetes & Istio를 이용한 AnyLink Master 모듈 (게이트웨이) 설계

Service Mesh Architecture의 학습 기간을 거쳤습니다.(2020.01 ~ 2020.03)

기존의 백엔드 서비스의 변화를 시도

  • 기존 Root Domain을 통한 중앙처리방식의 게이트웨이 아키텍처에서 Envoy Proxy를 이용한 Service Mesh 아키텍처로 재설계합니다.
  • Docker와 컨테이너의 개념, 그를 Orchestration하는 Kubernetes, 그리고 Istio 플랫폼과 Envoy Proxy까지 많은 것을 학습했습니다.
  • 팀 내 Kubernetes 클러스터 환경을 직접 구축하고 운영/관리하였지만, 다음 분기 팀 내부 사정에 의한 R&R변동때문에 해당 업무를 중지하게 됩니다..

EIMS 웹 백엔드 개발

EIMS(Enterprise Interface Management System)의 Web 백엔드 단을 개발하였습니다 (2020.02 ~ 2020.03)

한 번 더 백엔드 API 서비스 개발

  • 메시지 전문 및 인터페이스 정의, 거래, 배포 등을 통합관리 하는 EIMS를 설계 및 개발하였습니다.
  • 단말 및 단말서버, FEP/MCI 등에서 중복하여 개발해야 하는 전문 및 인터페이스를 한 번만 수행함으로서 개발 생산성 증대에 기여할 수 있습니다.
  • AnyLink 때와 마찬가지로 ProObject와 Java언어, DB는 Tibero 6를 사용하여 개발하였습니다.
  • DB 스키마 설계부터 메시지 전문 생성 및 정의 방식, 인터페이스 관리 및 배포 방식 등 바닥부터 설계 한 뒤, API 서비스를 개발하였습니다.
  • 1차 개발 완료 후, 2분기 팀 내 R&R 변동으로 인하여 인수인계한 작업입니다.
